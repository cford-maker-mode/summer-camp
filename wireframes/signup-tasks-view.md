## Auto-Creation Rules

Tasks are automatically created when:
1. A camp is added to the Summer Plan AND
2. The camp has a signup date defined

Tasks show the signup date from the camp record. If a camp has no signup date, no task is created (user can still manually track in the camp's notes).

---

## Design Decisions

- [x] Tasks auto-created when camps added to Summer Plan
- [x] Grouped by urgency (not alphabetical or by session date)
- [x] Overdue shown prominently at top
- [x] Mark Done can optionally update session to Confirmed
- [x] .ics export for external calendar integration (P1)

- [x] Child name shown on each task (supports multi-child families)
